# Changelog


## V0.9.11

- 新增 Windows CMD 打包入口，避免 PowerShell 未签名脚本执行策略拦截。
- 新增根目录一键打包 CMD 文件，可双击生成常用角色包。
- 正式插件包仍不包含脚本、说明文档和母包源码。


- 新增角色打包母包能力：通过脚本参数 `-Roles 1,2,3` 生成不同角色版本。
- 角色配置继续使用数字 1/2/3；生成包默认隐藏未启用角色，可通过参数置灰显示。
- 自动采集默认开启，仍支持按天、工作日、指定周几、按月第几号配置。
- 正式角色安装包不包含打包说明和 scripts 目录。

## V0.9.11

- 检测/探活读取页面状态改用 chrome.scripting，不再 attach chrome.debugger。
- 修复点击检测时浏览器顶部出现“已开始调试此浏览器”提示并造成窗口晃动的问题。
- 多 Profile 场景下，检测一个账号不会因为检测链路触发调试提示影响其他窗口。

# CHANGELOG

## V0.9.5

- 新增角色配置文件 `config/collector-role.config.json`。
- 新增三类角色按钮：客服-数据报表、财务-提现账单、财务-涉税账单。
- 自动采集计划改为按角色独立保存。
- 新增计划频率：每天、工作日、指定周几、每月第几号。
- 同步载体增加 `collectRole` 和 `collectRoleName`。
- 客服-数据报表继续复用现有采集链路。
- 财务-提现账单、财务-涉税账单完成入口和计划结构预留。
